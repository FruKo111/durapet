DuraPet — durapet.com.tr (Next.js) CSS yok / 404 / 403
=====================================================

GORDUGUN SEMPTOMLAR
- Sayfa metin olarak geliyor, stil yok  -> Tarayici `/_next/static/...css` dosyasini ya alamiyor (404) ya da HTML donuyor.
- `/sahip` vb. acilinca Hostinger kaykayli 404  -> Istek Node (next start)'a gitmiyor.
- `www` 403, apex 404  -> www ile apex farkli sanal host / farkli klasor.

BUNLAR KOD HATASI DEGIL — Hostinger on sunucu (LiteSpeed/Apache) ile Node arasindaki yonlendirme eksik veya yanlis.

"DAGITIM TAMAMLANDI" NE DEMEK?
- Hostinger'da yesil tik = genelde: repo cekildi, build komutu (bizde hostinger-build.js + next build) 0 koduyla bitti, paket kopyalandi.
- Bu, tarayicinin actigi HER istegin (ozellikle /_next/static/*.css ve /sahip gibi yollar) %100 Node'a ulastigi anlamina gelmez.
- HTML iskeleti bazen gelir (bir istek Node'a carpiyor), CSS/JS baska isteklerle gelir; onlar 404 olunca sayfa "css'siz" gorunur.
- Giris sonrasi 404: ya API hala HTML donuyor (yanlis API URL / eski bundle), ya da yonlendirme /sahip'e giderken yine on sunucu 404'u (Hostinger sayfasi) donuyor.
Yani: deploy geciyor diye "sunucu tarafi tamam" sonucu cikmaz; runtime yonlendirme ayri kontrol.

ILK YENILEMEDE CSS YOK, BIR YERE YAZIP ENTER DEYINCE DUZELIYOR — NEDEN?
- Ilk tam sayfa yuklemesinde tarayici HTML'i alir; CSS/JS ayri isteklerle `/_next/static/...` adreslerinden gelir.
- Bu istekler 404 olunca sayfa "sadece metin" gorunur.
- Sonra bir etkilesim (Enter, focus, client-side navigasyon) bazen farkli chunk'lari disk onbelleginden veya baska bir istekle yukler; kismen stilli gorunum olabilir — bu tutarli bir cozum degil, altyapi hatasinin belirtisi.
- Kalici cozum: tum `/_next/*` (ve panel rotalari) Node'a proxy; www/apex tek kanonik adres.

GIRIS: TOKEN 200, profilim 404 — AYNI MI?
- Hayir. `token?grant_type=password` (Ag'da gorunen) Supabase'e gider (`NEXT_PUBLIC_SUPABASE_URL`); bu yuzden 200 alinabilir.
- `profilim` ise DuraPet REST API'ye gider: tam adres `NEXT_PUBLIC_API_BASE_URL` + `/api/v1/profilim` olmali, ornek `https://durapet.site/api/v1/profilim`.
- Ag'da `https://durapet.com.tr/api/v1/profilim` goruyorsan: istek panele gidiyor; panelde bu route yok -> 404 + HTML. Cozum: hPanel ortaminda `NEXT_PUBLIC_API_BASE_URL=https://durapet.site`, sonra paneli YENIDEN DERLE (Next bu degeri build sirasinda bundle'a gomer).

SENIN YAPACAKLARIN (SIRAYLA)
1) Safari / Chrome: Sayfada sag tik -> Incekle -> Ag (Network).
   - Sayfayi yenile.
   - Tip: CSS veya "JS" filtrele.
   - Kirmizi olan `/_next/static/...` satirina tikla -> Status (404 mu? blocked mi?).

2) Adres tutarliligi
   - Tek bir kanonik adres sec: ya hep `https://durapet.com.tr` ya hep `https://www.durapet.com.tr`.
   - Digerini hPanel "Yonlendirme" ile 301 yap.
   - Panel + API CORS'ta kullandigin origin ile ayni olsun.

3) hPanel -> dosyalar -> ilgili domain `public_html` (veya Hostinger'in Node icin gosterdigi kok).
   - `.htaccess` var mi bak. Node uygulamasi genelde burada "proxy" kurallari ister.
   - PORT numarasi siteye ozel; Hostinger dokuman / destek ile dogrula.
   - ORNEK (PORT'u kendi Node portunla degistir — kopyala-yapistirma once destek onayi):

   <IfModule mod_rewrite.c>
   RewriteEngine On
   RewriteCond %{REQUEST_FILENAME} !-f
   RewriteCond %{REQUEST_FILENAME} !-d
   RewriteRule ^(.*)$ http://127.0.0.1:BU_PORTU_DEGISTIR/$1 [P,L]
   </IfModule>

   Not: `mod_proxy` acik olmali; paylasimli hostingte bazen sadece Hostinger otomatik yazar.

4) LiteSpeed / Cache aciksa: site icin "Purge" / onbellek temizle.

5) Destek mesaji (kopyala):
   "Next.js `next start` kullaniyorum. `/_next/static/` ve `/giris`, `/sahip` istekleri ya 404 ya da HTML; CSS yuklenmiyor. Apex ve www ayni Node uygulamasina bagli degil gibi. Tum path'lerin (ozellikle `/_next/*`) Node'a proxy edilmesi ve www+apex'in ayni siteye baglanmasi gerekiyor."

GERCEKCI ALTERNATIF
- Paneli Vercel / Netlify (ucretsiz katman) uzerinde barindirmak cogu zaman 5 dk surer; API `durapet.site` olarak kalir.
- `NEXT_PUBLIC_API_BASE_URL=https://durapet.site` + CORS'a panel origin ekle.
